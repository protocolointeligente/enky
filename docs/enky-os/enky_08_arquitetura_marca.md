# ENKY 08 - ARQUITETURA DA MARCA
**Versão 1.0** — Documento de organização da marca, módulos, subprodutos e nomenclatura

---

## Regra Central

> **Menos marcas. Mais clareza.** Modelo monolítico com módulos descritivos.

---

## Estrutura

```
ENKY (marca-mãe)
├── ENKY Platform (ecossistema completo)
├── ENKY Coach (experiência do treinador)
├── ENKY Athlete (experiência do atleta)
├── ENKY Intelligence (IA + análise + recomendação)
├── ENKY Analytics (dashboards + relatórios)
├── ENKY Marketplace (venda de planos/serviços)
├── ENKY Academy (educação + certificações)
├── ENKY Labs (pesquisa + inovação)
├── ENKY API (integrações)
└── ENKY OS (documentação interna + filosofia)
```

## Regras de Uso

1. ENKY sempre em destaque como marca principal
2. Subprodutos explicam função, não competem com marca-mãe
3. Não criar nomes sem necessidade comercial real
4. Evitar siglas internas no contato com usuário
5. Interface clara antes de sofisticada
6. Permitir expansão multiesportiva
7. Nenhum nome deve limitar a uma modalidade
8. Todo nome deve reforçar inteligência/clareza/ciência/performance

---

## Nomes a Evitar

- Nomes que limitem a corrida (ENKY Run)
- Nomes exagerados (ENKY Genius, ENKY Magic)
- Nomes que sugiram substituição (ENKY Robot Coach)
- Nomes confusos (ENKY Ultra AI Pro Max)
- Excesso de nomes parecidos para funções similares
