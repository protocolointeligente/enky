# ENKY 20 - ENKY INTELLIGENCE NO PRODUTO
**Versão 1.0** — Aplicação funcional da Intelligence em telas, fluxos e decisões

---

## Definição

> Camada distribuída de apoio à decisão, presente nos pontos certos do produto. **Não é chatbot genérico.**

5 funções: interpretar dados → explicar sinais → sugerir opções → gerar relatórios → apoiar decisões

---

## 12 Pontos de Entrada

| Onde | O que faz |
|------|-----------|
| **Dashboard treinador** | Atletas que precisam de atenção, resumo semanal |
| **Perfil do atleta** | Resumo inteligente, positivos, atenção, dados insuficientes |
| **Calendário** | Distribuição, sequência intensidade, conflitos, recuperação |
| **Prescrição manual** | Revisar coerência, alertar excesso, sugerir alternativa |
| **Geração automática** | Rascunho editável + justificativa + confiança + cautelas |
| **Periodização** | Sugerir fases, distribuição, regenerativas, polimento |
| **Feedbacks** | Interpretar RPE, dor, planejado vs realizado, tendência |
| **Análises** | Resumir tendências, explicar variações, padrões |
| **Relatórios** | Gerar relatório interpretativo (treinador=técnico, atleta=simples) |
| **Marketplace** | Revisar promessa, classificar nível, sugerir melhorias |
| **Dashboard atleta** | Resumo simples, lembrete, reforço positivo, sem alarmismo |
| **Admin** | Treinadores inativos, problemas, métricas de uso |

---

## Formato Padrão de Insight

1. **Observação** — o que foi detectado
2. **Interpretação** — por que importa
3. **Dados usados** — transparência
4. **Confiança** — baixa/moderada/alta
5. **Limitação** — o que falta
6. **Ação sugerida** — o que considerar

---

## Ações que NÃO pode executar sozinha

Cancelar treino, alterar carga sensível, mover sessão publicada, diagnosticar lesão, prescrever tratamento, bloquear atleta, enviar alerta alarmista, alterar marketplace publicado, mudar papel, alterar pagamento

---

## Logs Obrigatórios

Tipo de análise, usuário, atleta/plano, dados usados, resposta, confiança, ação sugerida, aceita/ignorada, edição do treinador, timestamp

---

## MVP Intelligence

Card de atenção (dashboard) + resumo no perfil + análise de feedback + planejado vs realizado + dados insuficientes + relatório simples + revisão de semana + logs + feedback do treinador sobre utilidade
